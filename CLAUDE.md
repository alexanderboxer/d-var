## Project Overview

d-var is a Hebrew Bible (Torah) text analysis tool. It combines external data sources (Sefaria texts, OpenScriptures Strong's dictionary) with hand-curated linguistic annotations to produce an interactive web interface for word-level exploration.

## Development Setup

- **Python 3.13+** with **uv** as the package manager
- Install: `uv sync`
- Run the CLI entry point: `uv run d-var`
- Run the dev script: `uv run python bin/dev.py`

There are no tests or linting configured.

## Architecture

**Data pipeline flow:** External sources → Parser (DataFrame) → Web generator (HTML/JS)

### Key modules (`src/d_var/`)

- **sources.py** — Downloads Torah texts from Sefaria and Strong's data from OpenScriptures GitHub repos into `data/external/`
- **parser.py** — Core logic. `torah()` builds a pandas DataFrame with every word across all five books, including consonantal/voweled/trope forms, Strong's numbers, lemmas, roots, and occurrence counts. `verse()` extracts a single verse.
- **templates.py** — Generates JSON template files (`data/templates/`) that are then hand-edited into `data/hand_parsed/auxiliary/` (clause breaks, Strong's-to-root mappings)
- **web.py** — `chapter_to_html()` generates static HTML pages and companion JS data files into `web/`
- **common.py** — `get_project_root()` utility

### Data directories

- `data/external/` — Downloaded source data (Sefaria JSON, OpenScriptures files). Regenerated by `get_external_files()`.
- `data/hand_parsed/auxiliary/` — Manually curated JSON: `clausebreaks.json` (verse clause divisions), `strongs2root.json` (Strong's number → Hebrew root mappings)
- `data/templates/` — Template files for bootstrapping hand-parsed data
- `web/` — Generated output: HTML chapter pages, CSS, JS (including large generated JSON data files like `genesis01.js`)

### DataFrame schema (from `torah()`)

Location: `idx`, `book`, `chapter`, `verse`, `word`, `clause`
Text forms: `d0` (consonants), `d1` (vowels), `d2` (vowels+tropes)
Linguistic: `strongs_number`, `strongs_lemma`, `lemma`, `root`
Metadata: `maqaf`, `paseq`, `break`
Counts: `d0_count`, `d1_count`, `d2_count`, `strongs_count`, `root_count`
